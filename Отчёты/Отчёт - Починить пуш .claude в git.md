# Отчёт — Починить пуш .claude в git

**Дата:** 2026-05-30 09:55
**Приоритет:** средний

---

## Диагностика

### Причина ошибки
Ошибка `08:56 | Bash | Check if .claude is a git repo and push ❌` возникает во время ежедневного цикла диагностики (`daily-self-dev.sh`, задача 3 — ШАГ 3).

Вложенный Claude Code, запущенный через PROMPT3, во время автономной диагностики решил проверить `~/.claude/` как git-репозиторий и попытался сделать push. Но `~/.claude/` — это **не git-репозиторий** (нет `.git/`), поэтому все git-команды вернули `exit 128` → `post-response-memory.sh` записал ❌ в рабочую память.

### Что проверено
- `~/.claude/` — нет `.git/`, это не репозиторий
- `session-stop.sh` — пушит только `axiom-void-website`, не `.claude`
- `post-edit-hook.sh` — пушит только сайт
- `post-response-memory.sh` — не делает push вообще
- `settings.json` — хуки только на сессию и PostToolUse

### Источник
`~/.claude/agents/daily-self-dev.sh`, PROMPT3 (Диагностика архитектуры), ШАГ 5.
Вложенный Claude Code самостоятельно решал проверить git во всех директориях, включая `~/.claude/`.

---

## Что сделано

Добавлен явный запрет в PROMPT3 после ШАГ 5:

```
ВАЖНО: ~/.claude/ — НЕ git-репозиторий. Не пытайся делать git init, git push
или любые git-операции в ~/.claude/. Это не нужно и будет ошибкой.
```

---

## Почему не инициализируем .claude как git-репо

`~/.claude/` содержит чувствительные данные:
- `.env` — токены Telegram, API-ключи
- `history.jsonl` — история всех сессий
- `projects/` — рабочая память с личными данными

Делать из этого публичный git-репозиторий — риск безопасности.
