# Выполни самодиагностику системы памяти и синхронизации.

## 🎯 Задача

Выполнить полную самодиагностику системы памяти и синхронизации: проверить целостность файлов BRAIN-INDEX.md, структуру CLAUDE.md, работоспособность записи на диск, граф связей между файлами — и немедленно исправить все найденные проблемы.

## 🤖 AI Dispatcher

Перед выполнением прочитай ~/.claude/agents/dispatcher.md и активируй нужных агентов:
Агенты для этой задачи: **Tech Lead | QA Tester | Researcher**
Tech Lead делает финальное ревью после всех агентов.

## 📋 Шаги выполнения

### ТЕСТ 1 — Проверка файлов памяти

1. Прочитай `~/.claude/CLAUDE.md` и найди путь к BRAIN-INDEX.md (ожидаемый: `~/vaults/Цифровой мозг/Brain/BRAIN-INDEX.md`)
2. Открой BRAIN-INDEX.md и извлеки полный список файлов из индекса
3. Для каждого файла в индексе выполни:
   - `ls -la <путь>` — проверить существование и дату изменения
   - `grep -c "Что сработало" <путь>` — наличие секции успехов
   - `grep -c "Что НЕ сработало" <путь>` — наличие секции провалов
   - `grep -o "\[\[.*\]\]" <путь> | wc -l` — количество wiki-ссылок
4. Сформируй таблицу результатов:

```
| Файл | Существует | Дата обновления | "Что сработало" | "Что НЕ сработало" | [[ссылки]] |
```

### ТЕСТ 2 — Проверка CLAUDE.md

5. Прочитай `~/.claude/CLAUDE.md` полностью
6. Проверь наличие всех 8 правил (пронумеруй найденные)
7. Для каждого правила проверь: `grep -n "БЕЗ ИСКЛЮЧЕНИЙ" ~/.claude/CLAUDE.md`
8. Выведи список: что присутствует, что отсутствует, где нет формулировки "БЕЗ ИСКЛЮЧЕНИЙ"

### ТЕСТ 3 — Проверка синхронизации с диском

9. Выбери первый существующий файл памяти из индекса
10. Запомни его текущий `mtime`: `stat <файл>`
11. Добавь в конец файла строку: `SYNC TEST — $(date "+%Y-%m-%d %H:%M:%S")`
12. Проверь обновление: `stat <файл>` — mtime должен измениться
13. Немедленно удали тестовую строку (последнюю строку файла)
14. Зафиксируй результат: ✅ запись работает / ❌ запись не работает

### ТЕСТ 4 — Проверка графа связей

15. Открой 3 файла из индекса (первый, средний, последний по списку)
16. Для каждого: `grep -o "\[\[.*\]\]" <файл>`
17. Если файл содержит 0 ссылок — добавь минимум 2 релевантных `[[ссылки]]` на другие файлы из индекса в секцию "Связанные файлы" (создай секцию если нет)
18. Сохрани изменения

### ПОСЛЕ ДИАГНОСТИКИ — Исправление проблем

19. Для каждого файла из Теста 1, где отсутствует секция "Что сработало":
    - Добавь секцию `## Что сработало\n> _Заполнить по итогам использования_`
20. Для каждого файла, где отсутствует секция "Что НЕ сработало":
    - Добавь секцию `## Что НЕ сработало\n> _Заполнить по итогам использования_`
21. Для каждого правила в CLAUDE.md без "БЕЗ ИСКЛЮЧЕНИЙ":
    - Добавь формулировку в конец правила
22. Добавь в CLAUDE.md новое правило:

```markdown
## Правило N — Еженедельная диагностика системы БЕЗ ИСКЛЮЧЕНИЙ
Каждый понедельник при первом запуске автоматически выполнять диагностику:
`~/vaults/Цифровой мозг/диагностика/system-health-check.md`
После выполнения — отправить краткий итог через Telegram бот.
```

23. Создай файл диагностики `~/vaults/Цифровой мозг/диагностика/system-health-check.md` — скопируй в него полный текст этого теста (все 4 теста + инструкции после диагностики)
24. Сформируй итоговый отчёт по структуре:
    - ✅ Что работает
    - ❌ Что не работает
    - ⚠️ Что работает, но требует внимания
    - 🔧 Что было исправлено в ходе диагностики
