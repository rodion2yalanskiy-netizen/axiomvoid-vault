# ТЗ: Анализ отчёта портфолио + настройка облачной работы

## Шаг 1 — Найти и проанализировать последний отчёт
- Открой папку /reports/ в Obsidian vault
- Найди последний отчёт об исправлении портфолио
- Проанализируй все ошибки и причины неудач
- Составь список: что не получилось и ПОЧЕМУ

## Шаг 2 — Выявить причины блокировок
Для каждой проблемы из отчёта определи:
- Не хватает прав доступа?
- Не хватает плагина или инструмента?
- Не хватает API ключа?
- Техническое ограничение системы?
Сохрани анализ в /reports/анализ-причин-[дата].md

## Шаг 3 — Установить необходимые инструменты
- Claude Code получает полный доступ на установку любых npm/pip пакетов
- Claude Code получает полный доступ на установку плагинов и инструментов
- Claude Code получает полный доступ на изменение конфигов
- Установи все недостающие инструменты необходимые для исправления ошибок из отчёта
- Список установленного сохрани в /reports/установленные-инструменты-[дата].md

## Шаг 4 — Исправить все ошибки из отчёта
- Пройдись по каждой проблеме из анализа
- Исправь каждую используя установленные инструменты
- После каждого исправления делай commit в GitHub

## Шаг 5 — ЗАПИСАТЬ В ПОСТОЯННУЮ ПАМЯТЬ Claude Code
Создай файл CLAUDE.md в корне репозитория со следующим содержимым:

---
# ПОСТОЯННЫЕ ПРАВИЛА — ЧИТАТЬ ПЕРЕД КАЖДОЙ ЗАДАЧЕЙ

## ОБЛАКО — ГЛАВНОЕ ПРАВИЛО
- ВСЕ процессы выполняются в облаке через GitHub Actions
- НИ ОДИН тяжёлый процесс НЕ сохраняется локально на MacBook
- MacBook используется ТОЛЬКО для написания заметок в Obsidian и git push
- Все файлы, отчёты, коды, агенты хранятся в GitHub репозитории

## АГЕНТЫ И АВТОМАТИЗАЦИИ
- Все агенты работают через GitHub Actions + OpenRouter
- Новые агенты создаются как .github/workflows/[имя].yml
- Все агенты имеют доступ друг к другу через GitHub репозиторий
- Агенты могут сотрудничать передавая данные через папки в vault

## РАЗРЕШЕНИЯ
- Claude Code имеет право устанавливать любые npm/pip пакеты
- Claude Code имеет право устанавливать любые плагины и инструменты
- Claude Code имеет право изменять любые конфиги в репозитории
- Claude Code имеет право создавать новые GitHub Actions workflows

## ОТЧЁТЫ
- Каждая выполненная задача = отчёт в /reports/
- Каждый отчёт = commit в GitHub
- Antigravity составляет финальный красивый отчёт после каждой задачи
---

## Шаг 6 — Финальный отчёт
- Antigravity составляет отчёт: что проанализировано, что установлено, что исправлено
- Сохранить в /reports/финал-портфолио-[дата].md
- Commit и push в GitHub
- НЕ сохранять ничего локально на MacBook

## Требования
- Все процессы в облаке GitHub Actions
- MacBook не нагружать
- CLAUDE.md записать в репозиторий как постоянная память
- Все агенты могут читать CLAUDE.md и работать по этим правилам
- Токены экономить: передавать только нужный контекст
