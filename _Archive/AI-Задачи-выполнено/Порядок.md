# 🧹 Навести полный порядок в хранилище

> **Для Claude Code.** Скопируй этот файл в корень хранилища Obsidian и запусти командой:
> `claude "Выполни все инструкции из файла '🧹 Навести порядок в хранилище.md'"`

-----

## Контекст и разрешения

Ты — Claude Code. Тебе поручено навести полный порядок в этом хранилище Obsidian.

**Владелец хранилища даёт полное разрешение на:**

- Удаление тестовых, пустых и мусорных заметок
- Удаление тестовых и черновых промптов
- Объединение дублирующегося контента
- Изменение frontmatter любых заметок
- Добавление и исправление вики-ссылок `[[...]]`
- Создание новых индексных и служебных заметок
- Реструктуризацию папок

**Нельзя:**

- Удалять папку `.obsidian/` и её содержимое
- Удалять папку `.git/` если она есть
- Изменять смысл и содержание заметок (только структуру и метаданные)

-----

## Шаг 1 — Полный аудит хранилища

Обойди рекурсивно все `.md`-файлы. Для каждого определи категорию:

```
АКТИВНАЯ    — есть содержательный текст, ссылки, или недавно изменялась
АРХИВНАЯ    — важный контент, но устарел или не используется
МУСОР       — пустая, одна строка, тестовая, "Untitled", "new note", "копия"
ТЕСТ/ПРОМПТ — тестовые промпты, черновики без ценности
ДУБЛИКАТ    — содержимое совпадает с другим файлом на 70%+
```

Выведи отчёт:

```
АУДИТ — [дата]
Всего файлов: N
  Активных:        N
  Архивных:        N
  Мусор:           N → [список файлов]
  Тест/промпты:    N → [список файлов]
  Дубликаты:       N → [пары файлов]
```

-----

## Шаг 2 — Очистка

### 2а. Удалить безвозвратно

Файлы категорий **Мусор** и **Тест/промпт**:

- Пустые файлы (0–3 строки без смысла)
- Файлы с названиями: `Untitled*`, `New note*`, `test*`, `tmp*`, `temp*`, `черновик*`, `копия*`, `draft*`
- Тестовые промпты и одноразовые инструкции

### 2б. Архивировать

Файлы категории **Архивная** и **Дубликат** — переместить в папку `_Archive/`:

- Дубликаты → `_Archive/_duplicates/` (перед удалением объединить уникальный контент в основной файл)
- Устаревшие → `_Archive/_old/`

### 2в. Записать лог

Создай файл `_CLEANUP_LOG.md` в корне хранилища:

```markdown
# Лог очистки — [дата и время]

## Удалено
| Файл | Причина |
|------|---------|
| path/to/file.md | пустой файл |

## Архивировано
| Файл | Куда | Причина |
|------|------|---------|
| path/to/file.md | _Archive/_old/ | нет ссылок, устарел |

## Объединено (дубликаты)
| Источник | Основной файл |
|----------|---------------|
| copy.md | original.md |
```

-----

## Шаг 3 — Восстановление и создание связей

### 3а. Найти незалинкованные упоминания

В каждой активной заметке найди слова/фразы, которые **точно совпадают** с названием другой заметки в хранилище, но ещё не обёрнуты в `[[...]]`. Добавь ссылки.

### 3б. Обновить frontmatter

Для каждой активной заметки добавь/обнови блок:

```yaml
---
updated: YYYY-MM-DD
tags: [определи по содержимому, 2–5 тегов]
related: ["[[Связанная заметка 1]]", "[[Связанная заметка 2]]"]
---
```

### 3в. Создать индексы для кластеров

Найди группы из 3+ заметок на одну тему. Если индексной заметки нет — создай `_INDEX/<Тема>.md`:

```markdown
---
tags: [index]
---
# <Тема>

## Заметки по теме
- [[Заметка 1]]
- [[Заметка 2]]
- [[Заметка 3]]
```

### 3г. Отчёт по связям

```
СВЯЗИ — итог
Добавлено новых [[ссылок]]:   N
Обновлено frontmatter:        N заметок
Создано индексных заметок:    N
Изолированных (нет связей):   N → [список]
```

-----

## Шаг 4 — Финальная карта хранилища

Создай или обнови файл `🗺️ Карта хранилища.md` в корне:

```markdown
---
tags: [index, map-of-content]
updated: YYYY-MM-DD
---

# 🗺️ Карта хранилища

## Структура
> Создана автоматически Claude Code [дата]

### 📁 Активные заметки по темам
- **[Тема 1]** — [[Индекс темы 1]] (N заметок)
- **[Тема 2]** — [[Индекс темы 2]] (N заметок)

### 📦 Архив
- `_Archive/` — N файлов

### 📊 Статистика
| Показатель | Значение |
|------------|----------|
| Всего активных заметок | N |
| Связей между заметками | N |
| Изолированных заметок  | N |
| Дата последней очистки | YYYY-MM-DD |
```
