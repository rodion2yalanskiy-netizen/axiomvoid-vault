# ТЗ: Обновление изображений на сайте

## Задача
Первые 5 изображений на сайте выглядят хорошо.
Все остальные изображения после первых пяти выглядят плохо и должны быть заменены.

## Шаг 1 — Antigravity: Анализ хороших изображений
- Открой сайт и найди первые 5 изображений
- Проанализируй их стиль, цвета, размер, композицию, настроение
- Составь подробное описание стиля для генерации новых изображений

## Шаг 2 — Antigravity: Генерация новых изображений
- На основе анализа первых 5 изображений сгенерируй новые изображения
- Новые изображения должны точно соответствовать стилю первых пяти
- Количество новых изображений = количество плохих изображений на сайте
- Сохрани все новые изображения в папку /assets/images/new/

## Шаг 3 — Antigravity → Claude Code: Передача изображений
- Antigravity передаёт Claude Code:
  - Путь к новым изображениям
  - Список плохих изображений которые нужно удалить
  - Структуру сайта где нужно сделать замену

## Шаг 4 — Claude Code: Обновление сайта
- Удали все плохие изображения (после первых пяти)
- Добавь новые сгенерированные изображения на их место
- Сохрани порядок и структуру сайта
- Сделай commit и push в GitHub

## Шаг 5 — Claude Code → Antigravity: Отчёт
- Claude Code сообщает Antigravity о результатах работы:
  - Сколько изображений удалено
  - Сколько новых изображений добавлено
  - Статус сайта после обновления
  - Ссылка на commit в GitHub

## Шаг 6 — Antigravity: Финальный отчёт
- Составь красивый отчёт в формате Markdown
- Включи: что было сделано, до/после, статистику, ссылки
- Сохрани отчёт в Obsidian vault в папку /reports/
- Название файла: Отчёт-обновление-изображений-[дата].md

## Требования
- Стиль новых изображений = стиль первых пяти
- Сайт после обновления не должен сломаться
- Все изменения через GitHub commit
- Токены экономить: передавать только нужный контекст
