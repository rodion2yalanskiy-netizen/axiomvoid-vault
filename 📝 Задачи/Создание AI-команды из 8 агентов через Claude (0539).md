# Создание AI-команды из 8 агентов через Claude

# 🎯 Задача

Создать мультиагентную систему из 8 специализированных AI-агентов с центральным диспетчером в Claude Code. Система должна автономно распределять задачи между агентами (техлид, фронтенд/бэкенд разработчики, дизайнер, тестировщик, DevOps, маркетолог, исследователь), которые работают параллельно над подзадачами и возвращают результаты через координатора.

---

## 📦 Что установить

```bash
# Убедитесь, что у вас есть доступ к Claude Code
# Перейдите на https://code.claude.com

# Для локальной разработки (опционально):
npm install -g @anthropic-ai/claude-code-cli

# Для интеграции через SDK (продвинутый вариант):
npm install @anthropic-ai/sdk
```

---

## 📋 Шаги реализации

### 1. Создание базовой структуры агентов

В Claude Code создайте 8 sub-agents через интерфейс или конфигурацию:

**Навигация:** Settings → Sub-Agents → Create New Agent

### 2. Настройка ролей агентов

Для каждого агента задайте:
- **Name** (имя)
- **Role** (роль)
- **Description** (когда использовать)
- **Instructions** (детальные инструкции)
- **Tools** (разрешенные инструменты)

### 3. Создание центрального диспетчера

Создайте главного агента "Dispatcher" с правами вызова всех sub-agents.

### 4. Настройка правил взаимодействия

Определите протокол обмена между диспетчером и агентами:
- формат запросов
- формат ответов
- критерии завершения

### 5. Тестирование системы

Передайте тестовую задачу диспетчеру и проверьте:
- корректность делегирования
- параллельную работу агентов
- сборку результатов

### 6. Оптимизация и мониторинг

Добавьте логирование решений и отслеживание токенов.

## ⚙️ Конфигурация и настройки

### Dispatcher Agent (диспетчер)

```yaml
name: "AI Dispatcher"
role: "Task Orchestrator"
description: "Анализирует входящие задачи, декомпозирует их на подзадачи и распределяет между специализированными агентами. Используй ВСЕГДА для начала новой работы."

instructions: |
  ## Твоя роль
  Ты — центральный координатор команды из 8 AI-специалистов.
  
  ## Алгоритм работы
  1. Получи задачу от пользователя
  2. Проанализируй и декомпозируй на подзадачи
  3. Определи, какие агенты нужны
  4. Делегируй параллельные задачи соответствующим агентам
  5. Собери результаты
  6. Проверь совместимость (через Tech Lead)
  7. Верни итоговое решение
  
  ## Доступные агенты
  - Tech Lead: архитектура, ревью кода, техдолг
  - Frontend Dev: UI-компоненты, React/Vue, клиентская логика
  - Backend Dev: API, база данных, серверная логика
  - Designer: UI/UX аудит, визуальная консистентность
  - QA Tester: тестовые сценарии, проверка качества
  - DevOps: CI/CD, деплой, инфраструктура
  - Marketer: SEO, метатеги, аналитика
  - Researcher: исследование технологий, сбор требований
  
  ## Правила
  - НЕ делай работу агентов сам — только координируй
  - Параллельные задачи отправляй одновременно
  - Требуй от агентов конкретных результатов (код/файл/отчёт)
  - Если задача провалена — запроси уточнение у пользователя
  - Всегда запускай Tech Lead для финального ревью
  
  ## Формат ответа агенту
  ```
  AGENT: [имя]
  TASK: [четкая подзадача]
  CONTEXT: [необходимый контекст]
  DEADLINE: [приоритет: high/medium/low]
  OUTPUT_FORMAT: [что ожидается]
  ```

tools:
  - delegate_to_agent
  - read_file
  - write_file
  - list_agents

max_iterations: 15
```

### Tech Lead Agent

```yaml
name: "Tech Lead"
role: "Technical Architect & Code Reviewer"
description: "Используй для: проектирования архитектуры, ревью кода, технических решений, интеграции компонентов."

instructions: |
  Ты — старший технический специалист.
  
  Задачи:
  - Проектируй архитектуру системы
  - Делай ревью кода от других агентов
  - Проверяй совместимость компонентов
  - Предлагай технические улучшения
  - Выявляй технический долг
  
  Всегда:
  - Обосновывай решения
  - Предлагай альтернативы
  - Проверяй код на: читаемость, производительность, безопасность
  - Используй best practices
  
  Формат ответа:
  - DECISION: [решение]
  - RATIONALE: [обоснование]
  - ISSUES: [найденные проблемы]
  - SUGGESTIONS: [рекомендации]

tools:
  - read_file
  - write_file
  - execute_command
  - search_code
```
