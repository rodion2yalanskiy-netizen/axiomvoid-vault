# Obsidian Sync — Содержимое папок не отображается на iPhone

**To:** support@obsidian.md  
**Subject:** Obsidian Sync: folder contents not showing on iOS — ✅ Отчёты folder appears empty

---

## Environment

- **Mac:** macOS 12.6.0 (Monterey), Obsidian 1.12.7
- **iPhone:** iOS (latest), Obsidian mobile (latest)
- **Plan:** Obsidian Sync (paid subscription)
- **Vaults:** 3 vaults — "Бизнес QSNera", "Цифровой мозг", "Личная жизнь"

---

## Problem Description

Files created on Mac in specific folders **do not appear on iPhone** even after successful sync. The folder itself is visible on iPhone, but when expanded it appears empty.

**Affected folder:** `✅ Отчёты` (inside vault "Бизнес QSNera")  
**Working folders:** Other folders in the same vault appear to sync correctly.

---

## Steps to Reproduce

1. On Mac: create a `.md` file inside `✅ Отчёты/` folder
2. Wait for Obsidian Sync to complete (green checkmark shown)
3. On iPhone: open same vault → navigate to `✅ Отчёты/`
4. **Expected:** file appears in folder
5. **Actual:** folder appears empty, no files visible

---

## What We Tried

1. **Disconnected and reconnected** Sync on both devices multiple times
2. **Checked sync.json** — `userId` field remained `-1` despite connecting through UI, indicating connection may not be persisting to disk
3. **Verified core-plugins.json** — Sync plugin is enabled (`"sync": true`)
4. **Checked allowTypes and allowSpecialFiles** — standard configuration, no custom exclusions
5. **Cleared vault and re-synced** — same result
6. **Tried creating aggregator vault** (mirroring 3 vaults into 1) — abandoned due to complexity
7. **Checked network** — WebSocket connections to sync-58.obsidian.md are established (ESTABLISHED state in lsof)
8. **sync-setup-auto plugin debugging** — plugin reports `userId=1` in runtime memory but sync.json on disk persists as `userId=-1`

---

## Key Technical Finding

After calling `syncPlugin.setup()` via internal plugin API:
- **In memory:** `syncPlugin.userId` returns `1` (appears connected)  
- **On disk (sync.json):** `userId` field remains `-1`  
- **Result:** Files are NOT syncing to remote vault

This suggests the sync connection is being established in runtime but not persisted to disk, causing it to be lost on next Obsidian restart.

---

## Suspected Causes

1. **Emoji in folder name** — `✅ Отчёты` starts with emoji ✅. Possible path encoding issue on iOS.
2. **Sync state not persisting** — `sync.json` `userId` stays at `-1` despite successful runtime connection, suggesting the sync state isn't being written to disk correctly.
3. **Selective sync exclusion** — The specific folder may be excluded by some default rule we're not aware of.

---

## Question

1. Why does `sync.json` keep `userId: -1` even after successful connection via Settings → Sync UI?
2. Are there known issues with folder names containing emoji (✅, 📝, etc.) on iOS sync?
3. Is there a way to force sync.json to be rewritten with correct connection state?

---

*Please advise. This is a business-critical workflow — reports generated by AI automation need to be accessible on mobile.*
